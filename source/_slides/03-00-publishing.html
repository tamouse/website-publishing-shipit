What do we mean by "Publishing"?

The act of uploading your website to a web host that can be reached by users.

{{page.title}}

How do you publish?

There are many ways to do this.

In this class, we're going to learn to use the File Transfer Protocol (FTP)

We'll be using a tool called "FileZilla" to perform our FTP operations

How does your site visitor find your website files?

{{page.title}}

index.html or index.php

When the user enters:
GDIMinneapolis.com/about/

The server looks for and returns
the first one of these it can find:

  • http://gdiminneapolis.com/about/index.php
  • http://gdiminneapolis.com/about/index.html

There are two things happening here:

  • The hostname that the user capitalized is getting shifted to lower case. All domains are lower case.

  • The server looks the index.php file first, and returns it, else it looks for the index.html file and returns it.

    In other words, the php file takes precedence over the html file if they both occur in the same directory.

{{page.title}}

public_html

  • On the remote server, you will publish your files to a folder called, public_html/.
  • This is the default file location for this hosting provider.

The public_html folder is the standard folder name on many web hosting providers, however, it's not the only possibility.

Make sure to check with your hosting provider to find out where you should upload your website files to.

Quite often, people forget to set the folder they should publish to during the transfer. In FileZilla, you can set the default folder that is opened with you connect to the site. You may want to set it to public_html/ to avoid such confusion.

File Transfer Protocol or FTP

  • We use FTP to transfer files from our local host (our own computer) to the remote server, at the hosting provider.
  • For this class we will use FileZilla to perform the FTP transfer.

Alternatives to FileZilla

  • Dreamweaver (integrated with the editor)
  • CyberDuck on macOS

The Filezilla UI is fairly easy once you get used to it. If you have never done anything like an FTP transfer before, though, it can be a little intimidating.

The QuickConnect area at the top lets you type straight in if you know your web hosting IP address, user name, and password.

You can also give the ftp address associated with your web hosting address, sometimes it's ftp.domainname.

{{page.title}}

When you've connected, you'll need to pay attention to the rest of the UI.

The top panel, under the quick connect bar, shows the messages going back and forth between the web server and your computer. Pay attention to any error messages that show up here.

The left hand side is for the local machine. Use the upper panel to navigate to the folder on your local machine containing the source of your web site.

The right hand side is for the remote machine. Use the upper pannel to navigate to the public_html folder to receive the source of your web site.

NOTE: you should be in the folder.

.

Select all the files and folders you want to transfer.

SPECIAL NOTE: do NOT select the parent folder indicated by the double dots.

Right-click the selection and choose the "start transfer" option. You can also find this in the menu bar at the top of the screen or window.

Watch the lowest pain that shows the transfers happening. If there are any errors during the transfer, they will show up in the "Failed Transfers" tab.

Let's Develop It

  1. Open Filezilla
  2. Enter your IP address into Host (199.167.200.163), and your username and password.
  3. QuickConnect: Are you connected?
  4. Find your local folder on the left side (you can navigate like a Finder or Explorer window)
  5. Find and open the public_html folder on the remote machine (right side)
  6. Upload your index.html file to the public_html folder
  7. Check to see if your files are live by going to your website domain address in the browser.
